# 小火箭焦虑拆解 — 项目总结文档

## 项目概述

「小火箭焦虑拆解」是一款 H5 互动小工具，帮助用户通过"写下焦虑 → AI 拆解 → 分类行动 → 释放放下"的流程，把模糊的焦虑变成具体、可控的行动碎片。视觉风格为像素风插画，整体氛围温暖但不鸡汤。

**线上地址**：https://xiaohuojian-eta.vercel.app

## 技术栈

| 层面 | 技术 |
|------|------|
| 框架 | React 19 + TypeScript |
| 构建 | Vite 7 |
| AI 模型 | 阿里云 DashScope（通义千问 qwen-turbo） |
| 部署 | Vercel（静态页面 + Serverless Functions） |
| 样式 | 纯 CSS（无 UI 库），像素字体 Fusion Pixel |
| 设计基准 | 750×1334 设计稿，前端 375×667 CSS px（资产 0.5x 缩放） |

## 页面流程（8 个屏幕状态）

```
loading → start → write → scan → assemble → sky → space → orbit
  P0       P1      P2      P3       P4        P5     P6      P7
```

### P0 Loading
百分比加载动画，小火箭三帧循环切图。

### P1 开始页
标题「焦虑拆解号」+ 小火箭（0.8x）+ START 按钮。左下角设置按钮（音乐/音效开关）和产品信息按钮（产品说明 + 免责声明 + BGM 致谢）。

### P2 写焦虑页
舱体掉落 → 点击舱门打开 → 纸卷展开 → 用户输入焦虑文本。底部有编辑和确认两个按钮（复用 P3 图标），纸卷本身不可点击关闭（防误触）。有新手蒙版说明。

### P3 扫描页
纸卷收入舱体上移进入扫描机。扫描光线动画 + BEEP 动效 + 进度条。AI 重述用户焦虑为"你在焦虑XX，因为希望XX"。三个按钮：修改（编辑 AI 总结）、确认（进入下一步）、重说（AI 重新生成，temperature=1.2）。有新手蒙版标注三个按钮功能。

### P4 组装页
传送带运送焦虑碎片（由 AI 从原始文本拆解而来）。用户将每个碎片分类为"我能行动"/"我能影响"/"我管不了"。支持编辑碎片文字。分类完成后 → 组件堆爆炸 → 火箭组装 → 发射。

### P5 建议云小游戏
天空场景，显示"我能行动"和"我能影响"的碎片。蓝色云朵（点击展开 AI 行动建议）+ 红色云朵（左滑消除焦虑碎片）。云组逐对出现，处理完毕后火箭飞入太空。有新手蒙版说明点击和滑动操作。

### P6 太空页
承接"我管不了"的碎片，以卫星形态在太空中漂浮。长按可定格查看 AI 给出的温暖释怀话语。有新手蒙版说明长按操作。

### P7 轨道页
星球 + 卫星环绕 + 三句诗意总结（AI 生成，长短不一）。温暖收尾。

## AI 集成

### 5 个 AI 调用

| 调用 | 触发时机 | 输入 | 输出 |
|------|---------|------|------|
| `aiRestate` | P3 扫描完成 | 用户原始文本 | 一句话重述（≤30字） |
| `aiSplit` | P3 确认后 | 原始文本 + AI 总结 | 3-4 个焦虑碎片（每个≤6字） |
| `aiAdvice` | P4 发射后 | 行动/影响类碎片 | 每个碎片的 tip（≤6字）+ detail（≤40字） |
| `aiAnalyze` | P4 发射后 | 释放类碎片 | 每个碎片的温暖释怀话（≤25字） |
| `aiSummarize` | P6 进入 P7 | 全流程上下文 | 三句诗意总结（长短不一） |

### AI 人设

所有 prompt 共享统一人设：
> 你是一个有洞察力的心理引导者，擅长透过表面描述看到深层情绪和需求。你的风格：温暖但不鸡汤，具体而不空洞，像一个聪明的朋友而不是机器人。

### 降级策略

每个 AI 调用都有 mock 数据兜底：API Key 未配置、网络错误、返回为空时自动降级到本地预设文案，确保流程不中断。

### 危机关键词检测

前端在提交文本前检测自杀/自残相关关键词，匹配时弹出温暖的危机干预界面，显示希望 24 热线（400-652-5580）。

## 新手蒙版系统

通过 `tutorialSeen` 状态记录每个页面是否已查看过教程。首次进入时自动弹出半透明黑色蒙版 + 白色提示文字，点击任意处关闭。

| 页面 | 提示内容 |
|------|---------|
| P2 写焦虑 | 写下你的焦虑 / 修改・确认 |
| P3 扫描 | 修改・确认・重说 |
| P5 天空 | 点击蓝色云朵查看建议 / 向左划红色云朵消除焦虑 |
| P6 太空 | 长按碎片可以定格查看 |

## 部署

### 本地开发
```bash
# 安装依赖
npm install

# 在 .env 中填入 API Key
DASHSCOPE_API_KEY=sk-xxxxxxxx

# 启动开发服务器
npm run dev
```

### 线上部署（Vercel）
1. 代码推送到 GitHub 仓库：https://github.com/fishwithoctopus/xiaohuojian
2. Vercel 自动检测 push 并部署
3. 环境变量 `DASHSCOPE_API_KEY` 在 Vercel 项目设置中配置
4. `vercel.json` 配置了 API 路由重写和 SPA fallback

### 架构说明
- **开发环境**：Vite dev server 内置中间件代理 `/api/chat` → DashScope API
- **生产环境**：Vercel Serverless Function（`api/chat.ts`）代理 AI 请求
- 两套代码的 prompt 保持同步

## 设计规范

- **设计稿尺寸**：750×1334px
- **前端基准**：375×667 CSS px
- **资产缩放**：统一 0.5x（即 viewBox 尺寸的一半）
- **字体**：Fusion Pixel（像素风）
- **主背景色**：`#f3ebd8`（米色）
- **重点颜色**：深蓝灰 `#3a4a5c`、暖红 `#c0392b`、金黄 `#e6a817`
